# Project Rules and Guidelines

## Overview
This document contains project-specific rules, guidelines, and standards that all agents must follow.

## Code Quality Standards
- Follow Go 1.26+ best practices
- Maintain consistent code style
- Write comprehensive unit tests
- Comment complex logic clearly

## Governance
- All major changes must be proposed as drafts in the 'drafts' directory
- Foundation documents are read-only and protected from direct edits
- Use the append_log tool for maintaining architectural decision records
